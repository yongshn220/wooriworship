프로젝트 전역에 대해 아래 웹앱 UX 설계 원칙 (UX Logic & Flow) 이 제대로 적용되어있는지 깊은 분석과 평가를 진행한 수 코드 변경하지말고 평가 내용을 보고하도록 해.


사용자의 인지 과정을 최적화하고 원활한 서비스 경험을 제공하기 위한 논리 설계 규칙: 

## 1. 인지 구조 최적화 (Information Architecture)
* **의사결정 단순화:** 사용자에게 한 번에 너무 많은 선택지를 주지 않는다. 필수적인 정보와 기능을 우선순위에 따라 노출한다.
* **정보 그룹화:** 연관된 정보는 논리적 단위로 묶어 제공함으로써 사용자가 시스템의 구조를 쉽게 파악하게 한다.
* **관습적 흐름:** 사용자가 이미 알고 있는 타 서비스의 일반적인 작동 방식(Mental Model)을 존중하여 설계한다.

## 2. 상호작용 및 피드백 (Interaction & Feedback)
* **즉각적 응답:** 사용자의 모든 행동(클릭, 입력 등)에 대해 시스템은 현재 상태를 알 수 있는 명확한 반응을 보여야 한다.
* **예측 가능성:** 특정 기능을 실행하기 전에 사용자가 그 결과를 예측할 수 있도록 충분한 맥락을 제공한다.
* **오류 관리:** 사용자의 실수를 시스템이 미리 방지하거나, 실수가 발생했을 때 사용자가 당황하지 않고 쉽게 되돌릴 수 있는 경로를 제공한다.

## 3. 맥락 유지 (Context Awareness)
* **단계별 정보 공개 (Progressive Disclosure):** 복잡한 정보는 사용자가 필요로 하는 시점에 맞추어 단계적으로 노출하여 혼란을 방지한다.
* **위치 자각:** 사용자가 서비스 내의 어느 단계에 있는지, 이전으로 돌아가려면 어떻게 해야 하는지 항상 명확히 인지하게 한다.

## 4. 접근성 및 보편성 (Universal Design)
* **조작 편의성:** 정교한 조작이 어려운 환경에서도 핵심 기능을 사용하는 데 지장이 없도록 동선을 설계한다.
* **기술적 배려:** 네트워크 속도나 기기 성능에 관계없이 사용자가 서비스의 핵심 가치를 빠르게 경험할 수 있도록 데이터 로드 순서를 최적화한다.
