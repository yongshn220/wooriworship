방금 업데이트한 코드에서 다음 6가지 원칙이 지켜졌는지 철저히 검수하고 코드 수정하지 말고 피드백을 줘.

# 6대 검수 원칙
1. Standardize (하드코딩 금지): 모든 수치는 디자인 토큰만 사용한다.
2. Componentize (부품화): UI는 잘게 쪼개어 공통 컴포넌트로 설계한다.
3. Decouple (로직 분리): UI와 로직을 섞지 않는다. 데이터 처리는 전용 Hooks/Service에서 수행한다.
4. DRY (중복 제거): 반복되는 로직은 무조건 공통 함수로 추출한다.
5. Define (타입 명시): 데이터 형태를 미리 정의하고 흐름을 명확히 밝힌다.
6. Modularize (파일 분리): 하나의 파일에 너무 많은 역할이 담기지 않도록 한다. 컴포넌트, 스타일, 타입, 로직(Hook)은 필요에 따라 물리적인 파일로 분리하여 관리한다.
